Montando tu primer laboratorio de hacking ético en casa

Nivel: Principiante · Tiempo de lectura: 15 min · Stack: VirtualBox + Kali + Metasploitable

Antes de lanzar cualquier exploit o escáner, necesitás un entorno seguro. Atacar sistemas sin permiso es ilegal y puede tener consecuencias graves. Un Home Lab es tu gimnasio personal: un lugar donde podés romper cosas, infectar máquinas y probar ataques sin riesgos legales ni técnicos para terceros.

1. Arquitectura del Laboratorio

El objetivo es simular una red real. Para esto, usaremos virtualización. Tu computadora física (Host) alojará dos o más computadoras virtuales (Guests).

  • Máquina Atacante: Será tu estación de trabajo. Usaremos Kali Linux, que ya viene con miles de herramientas preinstaladas.
  • Máquina Víctima: Un sistema vulnerable por diseño. Usaremos Metasploitable 2 o DVWA.

2. Herramientas Necesarias

Descarga el siguiente software (todo es gratuito):

  • VirtualBox: El hipervisor que correrá las máquinas virtuales. Descargar aquí.
  • Kali Linux VirtualBox Image: Una imagen lista para usar. No bajes la ISO si sos principiante, bajá la versión "VirtualBox".
  • Metasploitable 2: Una máquina Linux llena de agujeros de seguridad intencionales.

3. Configuración de Red (¡Muy Importante!)

Este es el paso donde la mayoría falla. Queremos que las máquinas virtuales se vean entre sí, y tengan internet para actualizarse, pero que estén aisladas de tu red doméstica (para no hackear accidentalmente la PC de tu familia).

La Red "NAT Network"

En VirtualBox, no uses "NAT" a secas (aísla las VMs entre sí) ni "Bridged" (expone las VMs a tu red real).

  1. En VirtualBox, andá a Archivo > Herramientas > Administrador de Red.
  2. Creá una nueva red llamada "NatNetwork".
  3. Habilitá el DHCP.
  4. En la configuración de CADA máquina virtual, andá a Red y seleccioná "Red NAT" (y elegí "NatNetwork").

4. Primeros Pasos con Kali Linux

Una vez que inicies Kali (usuario: kali, password: kali), abrí una terminal. Aquí están tus mejores amigos:

Nmap (Network Mapper)

Sirve para descubrir qué equipos hay en la red y qué puertos tienen abiertos.

# Escaneo rápido para descubrir hosts en la red
nmap -sn 10.0.2.0/24

# Escaneo de servicios y versiones a una IP específica
nmap -sV -sC 10.0.2.5

Netcat (nc)

La navaja suiza del networking. Sirve para leer y escribir datos a través de la red.

# Conectarse a un puerto específico
nc -v 10.0.2.5 80

5. Tu primera misión

1. Enciende Metasploitable 2.
2. Enciende Kali Linux.
3. Desde Kali, usa ifconfig para ver tu IP.
4. Usa nmap para encontrar la IP de Metasploitable.
5. Escanea los puertos de Metasploitable. ¿Ves el puerto 80 abierto? ¡Intenta abrir esa IP en el navegador Firefox de Kali!
