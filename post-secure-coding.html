Buenas prácticas para desarrolladores: Seguridad desde el diseño

Nivel: Todos · Tiempo de lectura: 10 min · DevSecOps

Históricamente, la seguridad era un obstáculo al final del desarrollo. Hoy, con DevOps y entregas continuas, la seguridad debe estar integrada ("Shift Left"). Aquí tienes los pilares fundamentales para escribir código seguro.

1. Validación de Entradas (Input Validation)

"Never trust user input". Todo dato que venga de fuera (formularios, APIs, parámetros URL) es potencialmente malicioso.

  • Whitelisting (Lista blanca): Acepta solo lo que sabes que es bueno. Ej: "El campo edad solo acepta números del 1 al 120".
  • Blacklisting (Lista negra): Intentar bloquear "lo malo" (ej. bloquear la palabra 'script') casi siempre falla porque los atacantes encuentran formas de evadirlo.
// Ejemplo de validación estricta (Node.js)
if (!/^[a-zA-Z0-9]+$/.test(username)) {
    throw new Error("Usuario inválido: solo letras y números.");
}

2. Prevención de XSS (Cross-Site Scripting)

XSS ocurre cuando un atacante logra ejecutar JavaScript en el navegador de otros usuarios. Para prevenirlo, debes usar Context-Aware Output Encoding.

Esto significa que antes de mostrar un dato en el HTML, debes convertir los caracteres especiales en sus entidades HTML seguras.

  • < se convierte en &lt;
  • > se convierte en &gt;
  • " se convierte en &quot;

Tip: Frameworks modernos como React, Vue o Angular hacen esto automáticamente por defecto. ¡No uses dangerouslySetInnerHTML o v-html a menos que sea estrictamente necesario!

3. Gestión de Secretos y Autenticación

El manejo de contraseñas y claves es crítico.

  • Nunca guardes contraseñas en texto plano. Usa algoritmos de hashing lentos y con salt, como Argon2 o Bcrypt. MD5 y SHA-1 están muertos.
  • No hardcodees credenciales. Las API Keys de AWS o de bases de datos no van en el código. Usa variables de entorno (.env).

4. Cabeceras de Seguridad (Security Headers)

Puedes endurecer tu aplicación web simplemente configurando las cabeceras HTTP correctas en tu servidor (Nginx, Apache, Express). 

# Ejemplo de cabeceras recomendadas
Strict-Transport-Security: max-age=31536000; includeSubDomains
Content-Security-Policy: default-src 'self'; script-src 'self' https://trusted.cdn.com
X-Content-Type-Options: nosniff
X-Frame-Options: DENY

5. Dependencias y Supply Chain

Tu código es solo el 10% de tu aplicación. El resto son librerías de terceros (npm, pip, maven).

Usa herramientas como npm audit, Snyk o OWASP Dependency Check en tu pipeline de CI/CD para detectar si estás usando librerías con vulnerabilidades conocidas.
